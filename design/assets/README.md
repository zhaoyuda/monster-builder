# 素材文件夹 · 上传说明

这个文件夹放游戏素材(零件图、图标、音效等)。上传后素材会自动跟着 wiki 一起发布,设计文档和试玩 demo 都能直接引用。

## 目录结构(按"批次 / 部位"两级分类)

```
assets/
  parts-side-v1/          ← 零件图第一批(侧视),画廊见 wiki「🎨 零件图」页
    head/    头 ×5
    torso/   躯干 ×5
    arm/     手/臂 ×5
    leg/     腿 ×5
    tail/    尾 ×5
    asset_manifest.json    每张图的代号清单
  (将来:parts-side-v2/、connectors/ 颈部连接件、ui/ 界面图标、audio/ 音效……)
```

- 上传零件图时**放进对应部位的子文件夹**(进到子文件夹里再点 Upload files);新批次开新的 `parts-side-v2/`,不覆盖旧图
- 图先传上来就行,和零件表的对应关系去 wiki「🎨 零件图」页的表格里填

## 怎么上传(GitHub 网页,不用装任何东西)

1. 打开这个文件夹的 GitHub 页面:仓库 → `design` → `assets`
2. 点右上角 **Add file → Upload files**
3. 把文件**拖进去**(可以一次拖多个,**整个文件夹拖进来也行**)
4. 底部随便写一句说明(比如"上传零件图第一批"),点绿色的 **Commit changes**

> 📦 **zip 压缩包请先在电脑上解压**,再把解压出来的文件拖进来——zip 直接传上来的话,网页上打不开也用不了。嫌麻烦就把 zip 直接发给 Yuda,他来处理。

上传成功后 Yuda 的手机会自动收到通知,不用另外说。

## 命名约定(重要,方便程序自动对接)

- **零件的图**:文件名 = 零件表里的零件名,比如 `新手头.png`、`猛爪.png`、`喷火头.png`
  - 这样将来 demo 里的怪兽就能按名字自动换上你画的图,一张都不用人工登记
- 其他素材(背景、图标、界面)随意起名,中文没问题
- 格式建议:零件图用 **PNG(透明背景)**;方形或接近方形最好搭,尺寸不用纠结,256×256 以上即可

## 注意

- 网页上传**单个文件最大 25MB**,图片音效都绰绰有余;真有超大的(视频、源文件)直接发给 Yuda
- ⚠️ 这个仓库是**公开**的,传上来全网可见——版权不明的参考图、还不想公开的草稿,先别传这里,发 Yuda 私下存
