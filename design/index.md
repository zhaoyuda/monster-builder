# 总览 · 手搓一个大怪兽

> 本 wiki 由 `design/` 目录下的 Markdown 文件自动构建。设计师 Akun 改 md → 提交 → 自动发布到这里。

「手搓一个大怪兽」设定书。核心玩法:**购买部件、装配怪兽、自动对战**,做出各种天马行空的怪兽造型。当前处于雏形阶段,设计边做边想。

## 系统地图

游戏由四大系统组成,优先级见标注:

| 系统 | 名称 | 简述 | 优先级 |
|---|---|---|---|
| **A** | 装配系统 | 购买怪兽部件并装配,怪兽由头、手、腿、躯干、插件五大类组成 | 核心 |
| **B** | 战斗系统 | 装好的怪兽进行回合制自动战斗 | 核心 |
| **C** | PVE 系统 | 难度逐渐提升的 PVE 流程 | 靠后 |
| **D** | PVP 系统 | 做好的怪兽进行对战(拟命名「怪兽围栏」) | 后续 |

## MVP 范围(第一个可玩版本)

**最小可玩闭环:**

> 搭怪兽 → 打 1 关 PVE(自动战斗看战报) → 赢了拿「吨」 → 买 / 换部件 → (可选)打下一关

| 纳入 MVP | 说明 |
|---|---|
| 系统 A 装配 | 部件、价格 / 功率约束、组装 |
| 系统 B 战斗 | 全自动战斗框架,详见 [系统B · 战斗](02-combat.md) |
| Ca 固定剧情 PVE | 1 关即可作为第一个可玩版本 |
| Cc 消化吸收 | 打赢获得「吨」,用于买 / 换部件 |

**暂不实现**(预留扩展点,见各页 Backlog):Cb 肉鸽、Cd 刷新、系统 D PVP、Ba 回合前策略。

### 子系统一览

**系统A · 装配** —— 详见 [系统A · 装配](01-assembly.md)

- **Aa. 价格系统**:确定身体组件的价格和功耗
- **Ab. 组件系统**:确定不同组件的属性

**系统B · 战斗** —— 详见 [系统B · 战斗](02-combat.md)

回合制战斗,每回合按 **先攻 → 各部位发起攻击(腿→手→头)→ 闪避判定 → 格挡判定 → 伤害结算** 的顺序进行(v2 已取消独立的「命中」公式,默认命中、仅由闪避减免;以 [系统B · 战斗](02-combat.md) 为准)。

- **Ba. 回合前战斗策略系统**:开战前选择战斗策略倾向(优先级靠后,*首期暂不做*)
- **Bb. 自动战斗系统**:根据怪兽特性进行战斗结算

**系统C · PVE**(优先级靠后) —— 详见 [系统C · PVE](03-pve.md)

- **Ca. 固定剧情 PVE**:战胜固定目标提升战斗力,熟悉战斗系统
- **Cb. 肉鸽 PVE**:按波次不停提升挑战目标的战斗力(*暂缓*)
- **Cc. 消化吸收系统**:战胜 PVE 敌人获得购买新零部件的资源
- **Cd. 零部件刷新系统**:每次刷新给出新可选零部件,刷新价格递增(*暂缓*)

**系统D · PVP**(*暂缓*) —— 详见 [系统D · PVP](04-pvp.md)

- **Da. 1V1 战斗**
- **Db. 3v3 KOF 战斗**
