# 系统B · 战斗系统

组装好的怪兽进行**全自动**战斗(无玩家输入)。

!!! info "本页以《战斗系统补充说明》为准"
    战斗框架以补充说明定稿为准;凡与最初设定书冲突或更细之处,以本页为准。**框架与规则已定,具体数值仍「待定」(占位)。**

战斗引擎应是一个**纯函数**,便于测试和复现:

```
battle(monsterA, monsterB, options) → BattleReport
```

## 战斗框架(定稿,引擎实现依据)

### 1. 怪兽 = 部件集合(关键)

一只怪兽**不是**「一坨汇总数值」,而是一组**各自独立的部件**,每个部件有独立属性和独立 HP。攻击、防御、命中、伤害都**按部件参与**,不按怪兽整体汇总。

### 2. 出手与回合

- 双方**轮流出手**(交替推进),不是双方同时一次性结算。
- **本回合谁先出手**:由重量 / 速度掷骰决定,轻 / 快者骰子更优(更可能先手)。
- 轮到某只怪兽出手时,它**所有存活的攻击型部件各进行一次攻击判定**——这是「多头多次咬人 / 多手多次出手」的来源。

!!! warning "取代最初设定书的 R1"
    本框架取代最初 R1「用 $N_1/(N_1+N_2)$ 概率决定谁先发动一次攻击」的描述,改为 **交替出手 + 掷骰定先手 + 每个攻击部件各自行动一次**。实现中如有歧义按此理解,并在代码注释里标 `TODO` 待确认。

    先手掷骰**具体公式属待定**,先用占位实现:**先手概率与速度正相关、与重量负相关**(权重数值待调)。

### 3. 单次攻击判定(沿用原 R2–R4)

每个攻击部件的**一次**攻击,依次计算:

| 步骤 | 规则 |
|---|---|
| **命中** | 基础 50%;攻击每高于防御 1 点 **+3%**,每低于 1 点 **−3%** |
| **暴击** | 攻击超过防御 **34 点**后,每点溢出 **+2%** 暴击概率,暴击造成**双倍伤害** |
| **闪避** | 命中后,最终命中率 = 命中率 ×(1 − 闪避) |
| **伤害** | 命中后由该攻击部件造成伤害;有减免则计算减免后伤害 |

$$
命中率 = 50\% \times (1 + (攻击 - 防御) \times 3\%) \quad,\quad 最终命中率 = 命中率 \times (1 - 闪避)
$$

### 4. 伤害落点

命中后,伤害**随机**落到目标的某个存活部件,扣减该部件 HP。(以后可能加「偏好落点」,现在随机即可,见 [Backlog](#backlog)。)

### 5. 部件掉落 / 降级

- 部件 HP 归零 → 该部件移除。
- 降级在**回合结束后**结算,下一回合按新形态计算,**不在回合内即时重算**。

### 6. 胜负条件

- **躯干 HP 归零 = 该怪兽失败**(躯干是目前唯一的「致命部位」)。
- 战斗结束:一方躯干爆,**或**达到关卡设定的回合上限(关卡可指定「限定回合内取胜」)。

## 算例(将来可作为单元测试用例)

!!! note "数值精确,适合在游戏代码中作为单元测试断言"

| 场景 | 计算 | 结果 |
|---|---|---|
| 70 攻 vs 40 防 | 50% × (1 + 30 × 3%) | **命中率 95%** |
| 40 攻 vs 70 防 | 50% × (1 − 30 × 3%) | **命中率 5%** |
| 暴击阈值 | 攻击 − 防御 > 34 点后,每点溢出 +2% 暴击 | 双倍伤害 |
| 闪避 | 命中率 × (1 − 闪避) | 闪避后最终命中率 |

## Backlog · 扩展点(暂不做,但代码结构要预留,别写死) {#backlog}

战斗引擎设计成可扩展,尤其:

- 在「**命中后 → 伤害前**」留一个可插入的阶段 hook,方便以后加格挡。
- **伤害结算**与「暴击 / 致命判定」**解耦**,方便以后加独立的致命一击机制。

待加功能:

- **嘲讽 + 概率格挡**:手作为拦截单位,在命中后、伤害前插入格挡判定;多只手分别计算。
- **致命一击 / 一击必杀致命部位**:独立于现有暴击的机制,拟按「智力」触发(智力来源待定,可能是头属性)。
- **多部件拦截率叠加**。
- **预设怪兽**(彩蛋角色 / 测试样本):无头怪、多头怪、三手嘲讽怪等。
- 后续系统:[Ba 回合前策略](#bb-自动战斗系统)、[Cb 肉鸽 / Cd 刷新](03-pve.md)、[系统 D PVP](04-pvp.md)。
