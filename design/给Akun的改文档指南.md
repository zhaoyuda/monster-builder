# 📝 给 Akun 的改文档指南

> 这份指南教你怎么在网页上改游戏设计文档。**全程在浏览器里点几下,不用装软件、不用敲命令、不会写代码也完全没问题。**
>
> 改完之后,网站会在一两分钟内自动更新。手机、电脑都能看:
> 👉 **https://zhaoyuda.github.io/monster-builder/**

---

## 第 0 步:只做一次的准备

1. 注册一个 GitHub 账号(github.com,跟注册普通网站一样)。
2. 把你的 GitHub 用户名发给 Yuda,他把你加进项目(给你编辑权限)。
3. 你的邮箱会收到一封邀请邮件,点里面的 **Accept(接受)** 就好。

> 做完这一步,你才能直接改文档。没做的话点了改也存不进去。

---

## 改文字(最常用)

1. 打开仓库:**https://github.com/zhaoyuda/monster-builder**
2. 点进 **`design`** 文件夹,再点开你要改的那个文件。每个文件管一块内容:

   | 文件 | 管什么 |
   |---|---|
   | `index.md` | 总览 / 系统地图 / MVP 范围 |
   | `01-assembly.md` | 系统A · 装配(部件、价格) |
   | `02-combat.md` | 系统B · 战斗 |
   | `03-pve.md` | 系统C · PVE |
   | `04-pvp.md` | 系统D · PVP |
   | `05-open-questions.md` | 待确认问题(在这里回答 Yuda 的提问) |

3. 点文件**右上角的铅笔图标 ✏️**(鼠标放上去会显示 "Edit this file")。
4. 直接在框里改文字,像改记事本一样。
5. 改好后,拉到页面**最下面**,点绿色按钮 **「Commit changes」**,弹出来的小窗再点一次 **「Commit changes」** 确认。
6. 完成!等 1–2 分钟,刷新上面的网站链接,就能看到最新版。

## 改表格 / 数值(你最关心的,有偷懒办法)

文档里的表格,在网页源码里长这样(用竖线 `|` 分隔):

```
| 组件 | 攻 | 血量 | 价格 |
|---|---|---|---|
| 普通头 | 130 | 50 | 200 |
| 普通手 | 50 | 20 | 100 |
```

手敲这些竖线很烦。**你可以继续在 Excel 里排表,然后一键转换:**

1. 在 Excel 里选中你的表格,复制(Ctrl+C)。
2. 打开这个免费网站:**https://tableconvert.com/excel-to-markdown**
3. 把表格粘贴进左边的框。
4. 右边会自动生成 markdown 格式的表格,点复制。
5. 回到 GitHub,把它粘贴到文档里对应位置,Commit。

> 这样你永远不用手敲竖线,Excel 怎么舒服怎么来,转一下贴进去就行。

## Markdown 小抄(就这么几个)

| 想要的效果 | 这样写 | 显示成 |
|---|---|---|
| 大标题 | `# 装配系统` | 一级标题 |
| 中标题 | `## 价格系统` | 二级标题 |
| 小标题 | `### 普通头` | 三级标题 |
| **加粗** | `**重要**` | **重要** |
| 列表 | 每行开头 `- ` | 圆点列表 |
| 标"还没定" | 直接写 `(待定)` 或 `TODO` | (待定) |

> 拿不准格式时,**照着文件里现有的写法改最省事**——比如要加一行表格,就复制上面一行改数字。

## 不用怕改坏

- 每次 Commit 都自动存了历史版本,**任何改动都能找回**,放心大胆改。
- 真改乱了,截个图发群里,Yuda 或助手能一键还原。

## 遇到不懂的

任何一步卡住了,**直接截图发到群里问**,不用自己死磕。
